## 📝 使い方

### 基本的な使用方法

```javascript
// コメントで言語を指定すると、その言語の構文ハイライトが適用されます

const htmlTemplate = /* html */ `
  <div class="container">
    <h1>タイトル</h1>
    <p>これはHTMLとしてハイライトされます</p>
  </div>
`;

const styles = /* css */ `
  .container {
    display: flex;
    justify-content: center;
  }
`;

const query = /* sql */ `
  SELECT * FROM users WHERE active = 1
`;
```

### TypeScript での使用

```typescript
const gqlQuery: string = /* graphql */ `
  query GetUser($id: ID!) {
    user(id: $id) {
      name
      email
    }
  }
`;
```

## 🚀 動作確認手順

### 方法 1: 開発モードで確認（推奨）

1. VS Code でこのプロジェクトを開く
2. **F5 キー**を押す（または「実行とデバッグ」→「Launch Extension」）
3. 新しいウィンドウが開く
4. `test-example.js` または `test-example.ts` を開く
5. テンプレート文字列内のコードがハイライトされていることを確認 ✨

### 方法 2: パッケージ化してインストール

```powershell
# VSCEをインストール（初回のみ）
npm install -g @vscode/vsce

# パッケージ化
vsce package

# インストール
# VS Codeで: Ctrl+Shift+P → "Extensions: Install from VSIX"
```

## 🎯 動作確認のポイント

開発ウィンドウで以下を確認してください：

1. **HTML のハイライト**

   - タグ名が色付けされているか
   - 属性名とその値が区別されているか
   - クラス名が認識されているか

2. **CSS のハイライト**

   - プロパティ名が色付けされているか
   - 値（色、サイズなど）が適切にハイライトされているか
   - セレクタが認識されているか

3. **SQL のハイライト**

   - キーワード（SELECT, FROM, WHERE）が強調されているか
   - テーブル名やカラム名が認識されているか

4. **その他の言語**
   - 各言語固有のキーワードが適切にハイライトされているか

## ✨ 特徴

- **簡単な使用法**: `/* 言語名 */` とコメントするだけ
- **豊富な対応言語**: 72 言語に対応
- **IDE 統合**: VS Code のネイティブ機能として動作
- **タイプセーフ**: TypeScript でも完全に動作
- **軽量**: 追加のランタイム依存なし

## 🔧 カスタマイズ

新しい言語を追加したい場合：

1. `build/languages.js` に言語定義を追加
2. `npm run build` を実行
3. `package.json` の `embeddedLanguages` に追加
4. 拡張機能を再読み込み

詳細は `TESTING.md` を参照してください。
